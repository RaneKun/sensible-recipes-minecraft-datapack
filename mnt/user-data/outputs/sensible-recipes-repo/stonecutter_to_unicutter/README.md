# Stonecutter to Unicutter

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](../LICENSE)
[![Minecraft](https://img.shields.io/badge/Minecraft-1.20.1-brightgreen.svg)](https://www.minecraft.net/)
[![Pack Format](https://img.shields.io/badge/Pack%20Format-15-blue.svg)](https://minecraft.wiki/w/Resource_pack)
[![Type](https://img.shields.io/badge/Type-Resource%20Pack-blue.svg)]()

A minimal resource pack with one job: rename the **Stonecutter** block to **Unicutter** in-game.

This rename exists to better reflect the expanded role of the stonecutter when used alongside the [Sensible Recipes](../sensible_recipes/) or [Sensible Recipes +](../sensible_recipes_plus/) datapacks — where the block is no longer just for stone, but a universal cutting and crafting station for wood, logs, and modded materials too.

---

## ✨ What It Does

- Renames `Stonecutter` → `Unicutter` everywhere in the game UI (inventory, tooltips, chat)
- Achieved via a single `en_us.json` language file override — no textures, no models, no other changes
- Completely safe to add or remove at any time without affecting worlds or saves

---

## 🔧 Compatibility

| Minecraft Version | Pack Format | Supported |
|---|---|---|
| 1.20 – 1.20.1 | 15 | ✅ Yes |

**Pairs with:**
- [Sensible Recipes](../sensible_recipes/) — vanilla datapack
- [Sensible Recipes +](../sensible_recipes_plus/) — modded datapack

This resource pack is purely cosmetic and works independently of any datapack. However, it is designed to complement the Sensible Recipes packs and will feel out of place without them.

---

## 📥 Installation

1. Download `stonecutter_to_unicutter.zip`
2. Place the zip file in your resource packs folder:
   ```
   .minecraft/resourcepacks/
   ```
3. In-game, go to **Options → Resource Packs** and move `stonecutter_to_unicutter` to the active side
4. Click **Done** — the block will now read "Unicutter" in-game
